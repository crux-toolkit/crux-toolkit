crux-analyze-matches

Description:

Given as input a sequence database and PSMs generate additional run-specific features and analyze with user-specified algorithm (e.g. Percolator).

Usage:

crux-analyze-matches  [options]  <psm folder>  <protein input>

Input:

  • <psm folder> – The name of folder in which all the psm result files are located.
  • <protein input > – The name of the file in fasta format or the directory containing the protein index from which to retrieve proteins and peptides.
Output:

    Returns a reranked peptide list with q-values in sqt file format.
Options:
  • [--algorithm <percolator | qvalue | none> ] – The analysis algorithm to use. Default = percolator
  • [--parameter-file <filename>] – A file containing command-line or additional parameters. See the parameter documentation page for details.
  • [--feature-file <string>] – Optional file in which to write the features. Default = none.
  • [--verbosity <int>] – Specify the verbosity of the current processes from 0-100. Default = 40.
Warning messages:
  • Fatal: The file "folder path name" does not exist (or is not readable, or is empty).
  • Fatal: The file "protein input filename" does not exist (or is not readable, or is empty).
